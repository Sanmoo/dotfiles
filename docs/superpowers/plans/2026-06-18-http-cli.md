# `http` CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a small Python CLI in `general/bin/http` that wraps `curl` with ergonomic flags, `{{VAR}}` template interpolation, and base-URL env-var resolution, plus a shell test in `tests/http-test.sh` that stubs `curl`.

**Architecture:** Single-file Python 3 stdlib script with a shebang. `parse_args` uses argparse with subcommands. `apply_template` is a pure function that does `{{K}}` → `V` substitution with strict validation. `build_curl_args` is the unit-testable core that produces a list of `curl` arguments. `format_curl_command` joins that list with `shlex.quote` for `--dry-run`. `main()` reads the body file (if any), runs the template pass, then either prints or `subprocess.run`s the curl call.

**Tech Stack:** Python 3 (stdlib only — `argparse`, `subprocess`, `shlex`, `urllib.parse`, `pathlib`), bash for tests.

## Global Constraints

These apply to every task. The values are copied verbatim from the design spec at `docs/superpowers/specs/2026-06-18-http-cli-design.md`.

- **Python 3 stdlib only.** No `requirements.txt`, no `pyproject.toml`, no third-party imports.
- **Shebang:** `#!/usr/bin/env python3`.
- **Methods supported:** `get`, `post`, `put`, `patch`, `delete` as subcommands. `get` does not emit `-X` (curl's default); all others emit `-X METHOD`.
- **Flag short forms:** `-B`, `-t`, `-H`, `-q`, `-f`, `-d`, `-v`, `-i`, `-n`, `-k`, `-L`, `-h`.
- **Template marker:** `{{NAME}}` — case-sensitive, no surrounding whitespace allowed inside the braces. Strict validation: undefined variable exits 2 with `error: undefined variable in template: {{K}}` on stderr. Empty value (`-v "K="`) is allowed.
- **Base URL precedence:** `-B` flag > `HTTP_BASE_URL` env > `BASE_URL` env. None → exit 2 with `error: no base URL provided (use -B or set HTTP_BASE_URL)`.
- **Path/URL assembly:** `<BASE_URL>/<PATH>` with duplicate slashes collapsed. Query params appended as `?k=v&k=v`; values URL-encoded via `urllib.parse.quote_plus`.
- **`-f` and `-d` are mutually exclusive.** Passing both is an error.
- **Auto Content-Type for `-f`:** when no `-H` declares a `Content-Type` header (case-insensitive prefix match), add `application/json` for `.json`/`.jsonc`, `application/xml` for `.xml`, `application/octet-stream` otherwise. With `-d`, no auto Content-Type.
- **Output behavior:** default executes curl; `-n`/`--dry-run` prints a single line, copy-pasteable (round-trips through `shlex.split`), does not execute.
- **Errors from the tool:** exit code 2, message on stderr. Errors from curl: propagate curl's exit code.
- **Test style:** match existing `tests/client-credentials-token-test.sh` — bash, `set -euo pipefail`, tmpdir, stub `curl` recording argv, `grep` assertions. Run with `bash tests/http-test.sh`.

```bash
#!/usr/bin/env bash
# Tests for general/bin/http. Stubs curl and asserts on the recorded argv.
set -euo pipefail

SCRIPT="$(cd "$(dirname "$0")/.." && pwd)/general/bin/http"

assert_contains() {
 # assert_contains <haystack_file> <needle> <message>
 local haystack="$1" needle="$2" message="$3"
 if ! grep -Fq -- "$needle" "$haystack"; then
  echo "FAIL: $message" >&2
  echo "  expected to find: $needle" >&2
  echo "  in:" >&2
  sed 's/^/    /' "$haystack" >&2
  exit 1
 fi
}

assert_not_contains() {
 local haystack="$1" needle="$2" message="$3"
 if grep -Fq -- "$needle" "$haystack"; then
  echo "FAIL: $message" >&2
  echo "  expected NOT to find: $needle" >&2
  echo "  in:" >&2
  sed 's/^/    /' "$haystack" >&2
  exit 1
 fi
}

run_http() {
 # run_http <args...>  ->  sets HTTP_CURL_ARGS to the recorded curl argv
 local tmpdir
 tmpdir="$(mktemp -d)"
 mkdir -p "$tmpdir/bin"
 cat >"$tmpdir/bin/curl" <<'STUB'
#!/usr/bin/env bash
printf '%s\n' "$@" > "$CURL_ARGS_FILE"
STUB
 chmod +x "$tmpdir/bin/curl"

 HTTP_CURL_ARGS="$tmpdir/curl.args"
 export CURL_ARGS_FILE="$HTTP_CURL_ARGS"
 PATH="$tmpdir/bin:$PATH" "$SCRIPT" "$@" >"$tmpdir/stdout" 2>"$tmpdir/stderr"
 HTTP_STDOUT="$(cat "$tmpdir/stdout")"
 HTTP_STDERR="$(cat "$tmpdir/stderr")"
 HTTP_TMPDIR="$tmpdir"
}

# ---------- Test 1: simplest post call ----------
echo "test 1: simplest post"
unset HTTP_BASE_URL BASE_URL
run_http post -B https://api.example.com foo
assert_contains "$HTTP_CURL_ARGS" "-X POST" "post should emit -X POST"
assert_contains "$HTTP_CURL_ARGS" "https://api.example.com/foo" "url should be base + path"
assert_not_contains "$HTTP_CURL_ARGS" "-X GET" "post should not include -X GET"

echo "OK"
```

- [ ] **Step 3: Create `general/bin/http` skeleton with just enough to make the test pass**

The skeleton implements `parse_args` (only `--base-url` and `--path` for now), `build_curl_args` (only method, base URL, path, slash dedup), `format_curl_command` (uses `shlex.quote`), and `main` (parses, builds, prints, exits 0). The script MUST be a real Python 3 file (not a stub) so future tasks can extend it. The functions introduced here will be extended in later tasks; their names and signatures are stable.

```python
#!/usr/bin/env python3
"""http — minimal REST client that builds and runs curl commands."""
import argparse
import shlex
import sys


def parse_args(argv):
    parser = argparse.ArgumentParser(prog="http", description=__doc__)
    sub = parser.add_subparsers(dest="method", required=True)
    for m in ("get", "post", "put", "patch", "delete"):
        sub.add_parser(m)
    parser.add_argument("-B", "--base-url", dest="base_url", default=None)
    parser.add_argument("path", nargs="?", default=None)
    # The remaining flags are added in later tasks.
    parser.add_argument("-t", "--token", dest="token", default=None)
    parser.add_argument("-H", "--header", dest="headers", action="append", default=[])
    parser.add_argument("-q", "--query", dest="queries", action="append", default=[])
    parser.add_argument("-f", "--file", dest="file", default=None)
    parser.add_argument("-d", "--data", dest="data", default=None)
    parser.add_argument("-v", "--var", dest="vars", action="append", default=[])
    parser.add_argument("-i", "--include", dest="include", action="store_true")
    parser.add_argument("-n", "--dry-run", dest="dry_run", action="store_true")
    parser.add_argument("-k", "--insecure", dest="insecure", action="store_true")
    parser.add_argument("-L", "--follow", dest="follow", action="store_true")
    return parser.parse_args(argv)


def build_curl_args(args, variables=None):
    # variables is unused in this task; full template pass is wired in Task 6.
    _ = variables
    out = []
    if args.method and args.method != "get":
        out += ["-X", args.method.upper()]
    if args.include:
        out.append("-i")
    if args.insecure:
        out.append("-k")
    if args.follow:
        out.append("-L")
    base = (args.base_url or "").rstrip("/")
    path = (args.path or "").lstrip("/")
    out.append(f"{base}/{path}" if (base or path) else "")
    return out


def format_curl_command(curl_args):
    return " ".join(shlex.quote(a) for a in curl_args)


def main(argv=None):
    args = parse_args(argv if argv is not None else sys.argv[1:])
    curl_args = build_curl_args(args)
    print(format_curl_command(curl_args))
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

## Task 5: Query parameters

**Files:**

- Modify: `general/bin/http` (extend `build_curl_args` to append `?k=v&k=v` to the URL).
- Modify: `tests/http-test.sh` (append tests 14–16).

**Interfaces:**

- `build_curl_args` must append `?` + URL-encoded key/value pairs (in user-given order) to the URL. Use `urllib.parse.quote_plus` on each value. Keys are NOT encoded (matches curl's `--data-urlencode key=value` style for query strings and avoids ugly `%2F` in keys).
- If no `-q` is given, the URL has no `?` suffix.

- [ ] **Step 1: Add query handling to `build_curl_args`**

Add the import at the top:

```python
from urllib.parse import quote_plus
```

Replace the URL assembly block in `build_curl_args` with:

```python
    base = (base_url or "").rstrip("/")
    path = (args.path or "").lstrip("/")
    url = f"{base}/{path}" if (base or path) else ""
    if args.queries:
        parts = []
        for q in args.queries:
            if "=" not in q:
                # Treat the whole string as a key with empty value.
                k, v = q, ""
            else:
                k, v = q.split("=", 1)
            parts.append(f"{k}={quote_plus(v)}")
        url = f"{url}?{'&'.join(parts)}"
    out.append(url)
```

```python
import re

_TEMPLATE_RE = re.compile(r"\{\{([^{}]+)\}\}")


def apply_template(text, variables):
    def repl(m):
        name = m.group(1)
        if name not in variables:
            print(
                f"error: undefined variable in template: {{{{{name}}}}}",
                file=sys.stderr,

            sys.exit(2)
        return variables[name]
```
